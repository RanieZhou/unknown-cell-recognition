# scGRN 当前实验总览（E005-E007）

## 1. 文档目的

这份文档用于整理当前仓库中已经完成的实验流程、关键脚本、可复用产物、核心结果与当前结论。

重点回答两个问题：

1. 从原始数据开始，当前最优且可复现的完整实验流程是什么。
2. 到 E007 为止，哪些方法真正有效，哪些假设已经被否定。

---

## 2. 当前结论先看

截至 E007，当前最值得作为“主结果”汇报的方法不是 E005 的 expression-only，也不是 E006 的 selective fusion，而是：

- `lineage_selective_rescue_globalT`

它来自 E007，但实际有效的是：

- lineage-aware rescue
- 全局阈值判定

而不是：

- bucket-specific threshold

当前主结论可以压缩成一句话：

> GRN 不适合作为全局增强信号，但在 expression 不确定、且主要位于 pDC-like corridor 的细胞上，作为局部 rescue 信号是有价值的；同时，ASDC 很可能不止是单一 pDC-like continuous unknown，而已经出现至少一个更偏 cDC1-like 的子模式。

## 3. 数据与任务定义

### 3.1 数据文件

主数据文件：

- `data/human_immune_health_atlas_dc.patched.h5ad`

当前默认假设：

- `adata.raw.X` 是 raw counts
- `var_names` 是 gene symbols
- `obs["AIFI_L2"]` 是标签列

### 3.2 当前任务设定

- Backbone: `scANVI`
- Dataset: `Immune DC`
- label column: `AIFI_L2`
- known classes: `cDC2`, `pDC`, `cDC1`
- unknown class: `ASDC`
- seed: `42`

### 3.3 数据划分

只对 known cells 做 train/val/test 划分，unknown 全部进入 test：

- `train_known`: 70%
- `val_known`: 15%
- `test_known`: 15%
- `test_unknown`: 全部 ASDC

训练时在 `train_known` 内随机 masking 10% 标签为 `unlabeled`，用于 scANVI 的半监督设置。

## 4. 从数据加载开始的完整实验流程

下面这条流程整理的是 E005-E007 的历史实验主线，主要用于理解结论来源与 legacy 产物。当前日常开发与维护应以 `src/scgrn/` 和根目录入口脚本为准。

### Step 1. scANVI expression backbone

脚本：

- `outputs/scripts/legacy/e005/run_e005_scanvi_backbone.py`

主要内容：

1. 读取 `h5ad`
2. 从 `adata.raw.X` 构建 `counts` layer
3. 固定 seed=42 做 known/unknown 划分
4. 在 `train_known` 中做 10% label masking
5. 先训练 `SCVI`
6. 再通过 `SCANVI.from_scvi_model()` 微调
7. 输出 expression 侧分数：
   - `entropy`
   - `distance_norm`
   - `expr_fused`
   - 每类概率 `prob_*`

关键产物：

- `outputs/E005/v1/E005_expression_scores.csv`

#### Step 1 结束后到底得到了什么

这一步结束后，真正可供后续使用的不是“一个训练好的模型”这么简单，而是每个 cell 的一组 expression 侧推断结果：

1. `predicted_label`
   - 来自 scANVI 的 hard prediction
   - 表示该 cell 在已知类别空间内最像哪个 known class

2. `prob_cDC2`, `prob_pDC`, `prob_cDC1`
   - 来自 scANVI 的 soft prediction
   - 表示该 cell 属于 3 个 known class 的分类概率

3. `entropy`
   - 用 soft prediction 计算
   - 公式：
   - `entropy = -sum_k p_k log(p_k)`
   - 含义：越高表示 expression 分类越不确定

4. `latent representation`
   - 由 `scanvi_model.get_latent_representation()` 得到
   - 这是后续做 expression distance 的基础空间

5. `nearest_known_class`
   - 对每个 cell，在 latent space 中计算其到 3 个 known class centroid 的距离
   - 取最小距离对应的类

6. `distance_norm`
   - 先在 `train_known` latent 上拟合各类 centroid
   - 再计算每个 cell 到最近 known centroid 的距离
   - 最后使用 `val_known` 的 min-max 统计量做归一化
   - 含义：越高表示越偏离已知表达分布

7. `expr_fused`
   - expression 侧主分数
   - 公式：
   - `expr_fused = 0.5 * entropy_norm + 0.5 * distance_norm`

换句话说，Step 1 的后处理实际上做了两件事：

- 用分类概率描述“分不清哪个 known”
- 用 latent 几何位置描述“离所有 known 有多远”

然后把两者融合成 `expr_fused`，作为 expression-only baseline。

### Step 4. GRN space scoring

脚本：

- `outputs/scripts/legacy/e005/run_e005_score_grn_space.py`

主要内容：

1. 读取 regulon activity matrix
2. 计算 known class 的 regulon centroids
3. 对所有 cells 计算 `grn_distance_score`
4. 输出 `nearest_grn_class`
5. 保存全量 `AUCell` 矩阵与 regulon centroid

关键产物：

- `outputs/E005/v2/E005_grn_scores.csv`
- `outputs/E005/v2/E005_aucell_all_cells.csv`
- `outputs/E005/v2/E005_regulon_centroids.csv`

#### Step 4 结束后到底得到了什么

这一阶段的关键不是“跑出了 regulon”，而是把所有细胞投影到了一个 regulon activity space 里，并且做成了可量化的 per-cell GRN 分数。

具体后处理步骤如下：

1. 读取 `AUCell` 矩阵
   - 行是 cell
   - 列是 regulon
   - 值是每个 cell 的 regulon activity

2. 在 `train_known` 上计算 3 个 known class 的 regulon centroid
   - `cDC2 centroid`
   - `pDC centroid`
   - `cDC1 centroid`

3. 对每个 cell，计算其到 3 个 centroid 的 cosine distance

4. 取最小距离作为：
   - `grn_distance_score`
   - 含义：如果一个 cell 在 regulon space 里离所有 known centroid 都远，则更像 unknown

5. 记录最小距离对应的类别作为：
   - `nearest_grn_class`

因此，Step 4 的输出本质上是：

- 一个 cell-level regulon embedding matrix
- 一个 GRN 侧异常程度分数
- 一个 GRN 侧最近 lineage 指示器

这两个信息在 E006/E007 中都会继续被用到。

### Step 5. E005 双分支融合基线

脚本：

- `outputs/scripts/legacy/e005/run_e005_dual_fusion.py`
- `outputs/scripts/legacy/e005/run_e005_v3_ml_fusion.py`

主要内容：

1. 保留 expression-only baseline
2. 比较 `dual_fused_v1`
3. 比较 `dual_fused_v2_rankavg`
4. 比较 `v3_ml_anomaly`

目的不是找复杂模型，而是回答：

- expression-only 是否已经很强
- GRN global fusion 是否真的增益

#### E005 的“推断后处理”具体怎么做

E005 在 expression 和 GRN 两个分支都已经得到 per-cell score 后，做的不是直接平均，而是先把分数统一到同一量纲，再做保守融合。

具体步骤：

1. 在 `val_known` 上，把 `expr_fused` 做 ECDF 标准化
   - 得到 `expr_pct`
   - 含义：当前 cell 的 expression 风险相当于 known validation 中的多少分位

2. 在 `val_known` 上，把 `grn_distance_score` 做 ECDF 标准化
   - 得到 `grn_pct`

3. E005 v1 保守融合
   - 公式：
   - `dual_fused_v1 = expr_pct + 0.2 * max(0, grn_pct - tau_g)`
   - 其中 `tau_g` 是 `val_known` 上 `grn_pct` 的 90% 分位数
   - 含义：GRN 只有在自己已经非常异常时，才允许给 expression 分数加成

4. E005 v2 rank-average
   - 公式：
   - `dual_fused_v2_rankavg = 0.7 * expr_pct + 0.3 * grn_pct`

5. E005 v3 anomaly
   - 使用 expression 特征 + top regulons 训练 Isolation Forest
   - 输出 `v3_ml_anomaly`

6. 在 `val_known` 上取 95% 分位数作为 unknown threshold
   - 每个方法都有自己的 threshold
   - 测试时 `score > threshold` 判为 unknown

所以，E005 的核心贡献不只是“做了几种 fusion”，而是系统验证了：

- 全局融合是否值得
- 如果不值得，问题出在 GRN 侧还是 threshold 侧

### Step 6. E006 selective fusion

脚本：

- `outputs/scripts/legacy/e006/run_e006_selective_grn.py`

主要内容：

1. 复用 E005 分数
2. 计算：
   - `expr_margin`
   - `grn_margin`
   - `grn_aux_score`
   - `expr_uncertainty_score`
3. 用固定 gate：
   - `gate_quantile = 0.90`
   - `alpha = 0.20`
4. 只在 expression 不确定区域引入 GRN
5. 做 unknown ratio ablation
6. 做 regulon interpretability 分析

核心回答：

- selective fusion 是否优于 expression-only
- GRN 是否更适合作为辅助信号，而不是全局增强信号

#### E006 的“推断后处理”具体怎么做

E006 的关键思想是：

> 不再对所有细胞统一融合，而是先判断 expression 是否不确定，只有不确定时才让 GRN 介入。

具体后处理步骤如下。

##### 6.1 Expression uncertainty 的构造

在 E005 的 expression 结果上新增：

1. `expr_margin`
   - `p_top1 - p_top2`
   - 这里 `p_top1` 是最大的已知类概率，`p_top2` 是第二大概率
   - margin 越小表示越难区分

2. `expr_margin_unknown_score_raw`
   - `1 - expr_margin`

3. 在 `val_known` 上拟合 min-max
   - `entropy -> entropy_minmax`
   - `expr_distance -> expr_distance_minmax`
   - `1 - expr_margin -> expr_margin_unknown_score`

4. 合成 expression uncertainty
   - `expr_uncertainty_score = (entropy_minmax + expr_distance_minmax + expr_margin_unknown_score) / 3`

##### 6.2 GRN 辅助分数的构造

E006 不再只用 `grn_distance_score`，而是同时考虑 regulon margin。

1. 对每个 cell，计算其到 3 个 known regulon centroid 的距离

2. 记：
   - `d1 = 最小距离`
   - `d2 = 第二小距离`

3. 定义：
   - `grn_margin = d2 - d1`
   - 含义：如果两个最近 known lineage 距离差很小，说明 GRN lineage 归属不明确

4. 在 `val_known` 上拟合 min-max：
   - `grn_distance -> grn_distance_minmax`
   - `grn_margin -> grn_margin_minmax`

5. 反向得到：
   - `grn_margin_unknown_score = 1 - grn_margin_minmax`

6. 最终：
   - `grn_aux_score = 0.5 * grn_distance_minmax + 0.5 * grn_margin_unknown_score`

##### 6.3 selective gate

固定先验参数：

- `gate_quantile = 0.90`
- `alpha = 0.20`

步骤：

1. 在 `val_known` 上取 `expr_uncertainty_score` 的 90% 分位数
   - 记为 `gate_threshold`

2. 对每个 cell：
   - 若 `expr_uncertainty_score < gate_threshold`
     - `selective_fused_score = expr_fused`
   - 否则
     - `selective_fused_score = 0.8 * expr_fused + 0.2 * grn_aux_score`

##### 6.4 阈值与评估

1. 在 `val_known` 上取 `selective_fused_score` 的 95% 分位数
   - 记为 `threshold_95`

2. 测试时：
   - `score > threshold_95` 判为 unknown

3. 同时保留 ranking 指标：
   - AUROC
   - AUPR
   - FPR95

##### 6.5 E006 的附加分析

推断后又额外做了三类分析：

1. unknown ratio ablation
   - 只改变 test set 的 known/unknown 组成比例
   - 不改变单细胞原始分数

2. error analysis
   - 按 TP / FP / TN / FN 分组
   - 比较 entropy、expr_distance、expr_margin、grn_distance、grn_margin、expr_uncertainty_score

3. interpretability
   - 比较 ASDC 与各 known class 的 regulon mean activity
   - 统计 ASDC 在 GRN 最近邻上主要靠近哪个 known class

### Step 7. E007 lineage-aware selective rescue

脚本：

- `outputs/scripts/legacy/e007/run_e007_lineage_selective_rescue.py`

辅助模块：

- `outputs/scripts/legacy/e007/e007_lineage_utils.py`
- `outputs/scripts/legacy/e007/e007_component_ablation.py`
- `outputs/scripts/legacy/e007/e007_asdc_analysis.py`

主要内容：

1. 直接复用 `outputs/E006/E006_scores_per_cell.csv`
2. 定义 `lineage_bucket`
   - `pDC_like`
   - `cDC1_like`
   - `cDC2_like`
3. 仅在特定 lineage corridor 上做 rescue
4. 对比两种阈值：
   - `lineage_selective_rescue_globalT`
   - `lineage_selective_rescue_bucketT`
5. 做 component ablation
6. 做 ASDC subgroup 分析

当前真正有效的版本是：

- `lineage_selective_rescue_globalT`

不是：

- `lineage_selective_rescue_bucketT`

#### E007 的“推断后处理”具体怎么做

E007 是在 E006 之上继续做推断后规则重构，不重新训练 backbone。

它解决的问题是：

> selective fusion 虽然有效，但不同 lineage corridor 的行为明显不同，是否可以只在更合理的 corridor 内做 rescue。

##### 7.1 先定义 lineage bucket

对于每个 cell：

1. `expr_lineage = predicted_label`
2. `grn_lineage = nearest_grn_class`

再定义：

- 若 `expr_lineage == pDC` 或 `grn_lineage == pDC`
  - `lineage_bucket = pDC_like`
- 否则若 `expr_lineage == cDC1` 或 `grn_lineage == cDC1`
  - `lineage_bucket = cDC1_like`
- 否则
  - `lineage_bucket = cDC2_like`

##### 7.2 rescue eligibility

固定参数：

- `alpha_pdc = 0.30`
- `alpha_cdc1 = 0.15`
- `alpha_cdc2 = 0.00`

首先仍然在 `val_known` 上对 `expr_uncertainty_score` 拟合 gate threshold。

然后对每个 cell 定义：

1. `is_bucket_gated`
   - `expr_uncertainty_score >= gate_threshold(bucket)`

2. `lineage_allowed`
   - 只允许 `pDC_like` 和 `cDC1_like`
   - `cDC2_like` 不允许 rescue

3. `is_positive_grn_rescue_candidate`
   - `grn_aux_score > expr_fused`
   - 即只有当 GRN 比 expression 更偏向 unknown 时，才允许 rescue

最终：

- `is_rescue_eligible = is_bucket_gated & lineage_allowed & is_positive_grn_rescue_candidate`

##### 7.3 E007 score 的构造

对每个 cell：

- 若 `pDC_like` 且 eligible
  - `score = expr_fused + 0.30 * (grn_aux_score - expr_fused)`
- 若 `cDC1_like` 且 eligible
  - `score = expr_fused + 0.15 * (grn_aux_score - expr_fused)`
- 否则
  - `score = expr_fused`

因此 rescue 是有方向性的：

- 只做向上拉升
- 不允许 GRN 把分数往回拉低
- 不允许 `cDC2_like` 走廊参与 rescue

##### 7.4 为什么 E007 有两个版本

score 本身在 `globalT` 和 `bucketT` 中其实相同，差别只在最终判定阈值。

1. `lineage_selective_rescue_globalT`
   - 在全部 `val_known` 上取一个全局 95% 分位数阈值

2. `lineage_selective_rescue_bucketT`
   - 对 `pDC_like / cDC1_like / cDC2_like` 分别在各自 `val_known` 上取 95% 分位数阈值

所以：

- `globalT` 和 `bucketT` 的 ranking 指标可以完全相同
- 但 classification 指标可能差很多
- 这正是 E007 结果里出现“AUROC/AUPR 一样，但 Macro-F1 差很多”的原因

##### 7.5 E007 的后续分析

E007 推断后又额外做了三部分分析：

1. bucket metrics
   - 在 `pDC_like / cDC1_like / cDC2_like` 内分别比较
   - unknown recall
   - known FP rate
   - rescued / hurt / net_gain

2. component ablation
   - 比较哪些组件真的起作用
   - 例如：
   - `no_bucket_threshold`
   - `no_cDC2_block`
   - `no_positive_only`

3. ASDC subgroup analysis
   - `always_TP_ASDC`
   - `rescued_ASDC`
   - `still_FN_ASDC`
   - 再比较这三组的 regulon activity 差异

因此，E007 的重点已经不只是分数更高，而是：

- 哪个 corridor 真能 rescue
- 哪个 corridor 会误伤 known
- 剩余 FN 到底是什么类型的 ASDC

### 6.0 一句话理解每一步产物是怎么串起来的

如果只看“推断后处理链条”，可以简单理解为：

1. **scANVI**
   - 负责输出 expression probability 和 latent embedding

2. **expression scoring**
   - 从 probability 得到 `entropy`、`expr_margin`
   - 从 latent 得到 `expr_distance`
   - 合成 `expr_fused`

3. **pySCENIC + AUCell**
   - 负责输出每个 cell 的 regulon activity

4. **GRN scoring**
   - 从 regulon activity 得到 `grn_distance`、`grn_margin`
   - 合成 `grn_aux_score`

5. **E006 selective fusion**
   - 利用 `expr_uncertainty_score` 判断哪些 cell 允许 GRN 介入

6. **E007 lineage-aware rescue**
   - 利用 `lineage_bucket` 决定哪些 corridor 可以 rescue
   - 利用全局阈值做最终 unknown 判定

## 7. 结果总览

### 7.1 E005：expression-only 已经很强

E005 关键结果：

- `expr_fused`: AUROC `0.9423`, AUPR `0.7444`, FPR95 `0.2387`
- `dual_fused_v1`: AUROC `0.9392`, AUPR `0.6616`
- `dual_fused_v2_rankavg`: AUROC `0.9267`, AUPR `0.5917`
- `v3_ml_anomaly`: AUROC `0.7661`, AUPR `0.3107`
- `grn_distance`: AUROC `0.7328`, AUPR `0.2555`

结论：

1. expression-only baseline 已经很强。
2. GRN 单独做 unknown detection 明显不够强。
3. 全局融合会稀释 expression 信号。

### 7.2 E006：selective fusion 有小幅稳定收益

E006 主结果：

- `expr_fused`: AUROC `0.9423`, AUPR `0.7444`, Macro-F1 `0.8086`
- `selective_fused_score`: AUROC `0.9430`, AUPR `0.7469`, Macro-F1 `0.8186`

gate 行为：

- `gated_fraction = 0.1552`
- `gated_fraction_on_unknown = 0.7318`
- `gated_fraction_on_known = 0.0671`

E006 结论：

1. selective fusion 比 expression-only 有小幅但真实的收益。
2. gate 的确命中了 expression 不确定区域。
3. GRN 更像局部辅助证据，而不是全局增强信号。
4. ASDC 在 GRN 空间主要靠近 pDC。

### 7.3 E007：lineage-aware rescue 有效，但 bucket threshold 失败

E007 主结果：

- `selective_fused_score`: AUROC `0.9430`, AUPR `0.7469`, Macro-F1 `0.8186`
- `lineage_selective_rescue_globalT`: AUROC `0.9450`, AUPR `0.7623`, Macro-F1 `0.8224`
- `lineage_selective_rescue_bucketT`: AUROC `0.9450`, AUPR `0.7623`, Macro-F1 `0.7926`

这一轮最重要的事实是：

- `globalT` 明显优于 E006
- `bucketT` 排序更好，但分类阈值决策更差

也就是说：

> E007 成功的是 rescue 规则，不是 bucket-specific threshold。

## 9. E007 的关键机制结论

### 9.1 pDC-like corridor 是主要收益来源

在 `pDC_like` bucket 中：

- `expr_fused` unknown recall = `0.7258`
- `selective_fused_score` = `0.7231`
- `lineage_selective_rescue_globalT` = `0.7876`

说明 E007 的主要收益确实来自 `pDC_like` corridor。

### 9.2 cDC2-like block 是必要的

在 `cDC2_like` bucket 中：

- E006 selective 的 FP rate = `0.0310`
- E007 globalT 的 FP rate = `0.0190`

component ablation 显示：

- `E007_no_cDC2_block` 的 `net_gain = -29`

说明：

> 禁止 `cDC2_like` rescue 是正确的。

### 9.3 bucket-specific threshold 不适合当前数据

component ablation：

- `E007_no_bucket_threshold`: `net_gain = +15`
- `E007_full`（bucket threshold）: `net_gain = -21`

尤其在 `cDC1_like` bucket 中，bucket threshold 导致大量 unknown 被重新压回 negative，严重损害 recall。

说明：

> 当前数据支持 lineage-aware rescue，但不支持 lineage-aware threshold。

### 9.4 positive-only rescue 不是当前主矛盾

component ablation：

- `E007_full`: `net_gain = -21`
- `E007_no_positive_only`: `net_gain = -18`

两者都远差于 `no_bucket_threshold`。

说明当前最关键的问题不是 positive-only，而是 threshold 选择。

## 10. 生物学解释：ASDC 到底像什么

### 10.1 E006 结果

ASDC 在 GRN 最近邻上：

- pDC: `71.07%`
- cDC1: `26.63%`
- cDC2: `2.30%`

这说明 ASDC 整体上明显偏 pDC-like。

### 10.2 E007 的新发现

E007 的 ASDC subgroup 分析显示：

- `always_TP_ASDC = 277`
- `rescued_ASDC = 22`
- `still_FN_ASDC = 223`

其中：

- `rescued_ASDC` 最近 GRN 类别主要是 pDC (`90.91%`)
- `still_FN_ASDC` 最近 GRN 类别主要变成 cDC1 (`57.40%`)

这比 E006 更进一步，说明：

1. ASDC 不是简单的“全都 pDC-like”
2. 至少有一部分 ASDC 更偏 `cDC1-like`
3. 当前 rescue 规则更擅长抓住 `pDC-like ASDC`
4. 对 `cDC1-like ASDC` 还缺少有效策略

当前最合理的解释是：

> ASDC 很可能已经不是单一 continuous unknown，而是至少开始呈现两个子模式：一个更偏 pDC-like，另一个更偏 cDC1-like。

## 11. 当前最稳的结论

到 E007 为止，可以稳定写进总结的结论有：

1. `expr_fused` 是非常强的 baseline。
2. GRN 单独做 unknown detection 区分力不足。
3. 全局融合、rank-average、Isolation Forest 都不如 expression-only。
4. selective fusion 是合理方向，因为它只在 expression 不确定区域使用 GRN。
5. lineage-aware rescue 比 E006 selective fusion 更进一步，但真正有效的是 `global threshold` 版本。
6. `cDC2_like` 不应该使用 rescue。
7. 当前收益主要来自 `pDC_like` corridor。
8. ASDC 在生物学上整体偏 pDC-like，但已经出现更偏 cDC1-like 的残余模式。

## 12. 当前推荐的汇报口径

如果需要一句话总结当前阶段工作，建议写：

> 在 Immune DC 数据上，scANVI expression-only 已经提供了强 unknown detection 基线；GRN 作为全局增强信号无效，但在 expression 不确定且主要位于 pDC-like corridor 的细胞上，作为 lineage-aware 局部 rescue 信号可以稳定带来增益。进一步分析表明，ASDC 整体偏 pDC-like，但残余漏检群体明显向 cDC1-like 偏移，提示 ASDC 可能已出现至少两个子模式。
