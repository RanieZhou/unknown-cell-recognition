# E005：scANVI + GRN on Immune DC

## 1. 实验目标

这轮实验只回答两个问题：

1. **在 Immune DC 数据集上，scANVI expression-only 是否已经能稳定识别 unknown。**
2. **加入 GRN 证据后，性能是否至少非劣，同时能提供更强的生物学解释。**

这里不追求一开始就“显著提升 AUROC”，而是先验证：

> **GRN 作为第二证据空间，能否在不明显拖累 unknown detection 的前提下，为拒识结果提供 regulon/TF 层解释。**

scANVI 本身是半监督单细胞注释模型，可以利用部分已知标签去推断其余细胞状态，并且可以显式接入 batch/covariate 信息；官方 API 也支持通过 `setup_anndata()` 注册 `labels_key`、`unlabeled_category` 和 `batch_key`，以及从预训练的 SCVI 初始化 `SCANVI.from_scvi_model()`。([Project name not set][1])

## 3. 数据划分

## 3.1 固定随机种子

```text
seed = 42
```

## 3.2 划分规则

只对 **known cells** 做 train / val / test 划分；**unknown 只进入 test**。

具体如下：

* `train_known`: 70%
* `val_known`: 15%
* `test_known`: 15%
* `test_unknown`: 全部 ASDC

## 3.3 为什么这样切

这是最干净的 open-set 评估方式：

* 训练阶段不暴露 unknown
* 验证阶段只看 known，用来做阈值与归一化
* 测试阶段再同时评估 known / unknown

## 3.4 训练时的半监督处理

因为你现在选的是 scANVI，而不是纯 SCVI，我建议在 `train_known` 内再做一层**标签掩蔽**：

* 从 `train_known` 中随机抽 **10% known cells**
* 将其标签改为 `"unlabeled"`

这样做的目的不是制造 unknown，而是让 scANVI 以它设计的半监督方式工作，更贴合其原始建模设定。scANVI 官方就是围绕“部分标签已知、部分未标注”的场景设计的。([Project name not set][1])

## 4. scANVI expression backbone

## 4.1 输入矩阵

对当前数据，直接使用：

```python
adata.layers["counts"] = adata.raw.X.copy()
```

然后 `adata.X` 可以保留现有 log-normalized 表达，用于可视化，但 scvi/scANVI 注册时明确指定：

```python
layer="counts"
```

因为 scvi-tools 官方明确强调：即使后续要做 normalize / log1p，也应保留 count 信息供模型使用。([Project name not set][2])

## 4.2 AnnData 注册

建议：

```python
SCVI.setup_anndata(
    adata_train,
    layer="counts",
    batch_key="batch_id",
    labels_key="scanvi_label",
)
```

然后：

```python
scvi_model = SCVI(...)
scanvi_model = SCANVI.from_scvi_model(
    scvi_model,
    unlabeled_category="unlabeled",
    labels_key="scanvi_label",
)
```

这符合 scvi-tools 当前 API 逻辑：先训练 SCVI，再用 `SCANVI.from_scvi_model()` 初始化 scANVI。([Project name not set][3])

## 4.3 训练配置

第一轮不要太激进，先求稳：

* `n_latent = 10`
* `n_layers = 2`
* `n_hidden = 128`
* `dropout_rate = 0.1`
* `gene_likelihood = "nb"`

训练策略：

1. **SCVI pretrain**

   * `max_epochs = 200`
   * 开启 early stopping

2. **scANVI fine-tune**

   * `max_epochs = 100`
   * 同样 early stopping

## 4.4 expression-only 输出

训练完成后，为每个 cell 取：

1. **分类概率**

   * `p(y|x)` over known classes

2. **entropy score**

   * `H(x) = - Σ p_k log p_k`

3. **latent embedding**

   * 用 `scanvi_model.get_latent_representation()`

4. **expression distance score**

   * 在 latent space 中，以 `train_known` 为参考，计算 cell 到各已知类原型的最小距离

---

## 5. expression-only unknown score

我建议你第一轮不要把 expression score 设计得太复杂，直接用两项：

## 5.1 entropy

高 entropy = 更像 unknown

## 5.2 latent distance

流程：

1. 在 `train_known` latent embedding 上，按类计算 centroid
2. 对每个测试细胞，算到各 centroid 的距离
3. 取 **最小距离** 作为 `expr_distance_score`

距离先用：

* **Mahalanobis distance**（首选）
* 若协方差不稳，则退化为 **Euclidean distance**

## 5.3 expression fused score

先在 `val_known` 上做 min-max normalization，再融合：

```text
expr_fused = 0.5 * entropy_norm + 0.5 * distance_norm
```

第一轮就固定 0.5/0.5，不学权重。
原因很简单：你的 `val` 里只有 known，没有 unknown，没法稳定学判别权重；固定权重更干净。

---

## 6. GRN / regulon branch

## 6.1 输入来源

只用：

```text
train_known
```

去构建 GRN。
不要把 unknown 混进 GRN 推断里，避免信息泄漏。

## 6.2 pySCENIC 输入

从 `train_known` 导出：

* 表达矩阵：`genes × cells` TSV
* TF list：`allTFs_hg38.txt`
* ranking db：你的 `hg38 ... feather`
* motif annotation：`motifs-v10nr...hgnc...tbl`

pySCENIC 官方 CLI 的标准流程就是 `grn -> ctx -> aucell`，其中：

* `grn`：从表达矩阵推导共表达模块
* `ctx`：做 motif enrichment / pruning
* `aucell`：得到 regulon activity matrix。([pySCENIC][4])

## 6.3 GRN branch 输出

拿到 regulon AUC matrix 后：

1. 在 `train_known` 上按类计算 **regulon centroid**
2. 对 `val_known / test_known / test_unknown` 中每个 cell：

   * 投影到 regulon activity space
   * 计算到各类 regulon centroid 的距离
3. 取最小距离作为：

```text
grn_distance_score
```

建议先用：

* **cosine distance** 作为主分数
  因为 regulon AUC 更像“程序活性谱”，方向信息通常比绝对量级更重要。

---

## 7. 融合方案

这里我不给你“平均融合”那种容易稀释表达主干的方案。
第一轮直接用一个**保守融合**，目标就是：**尽量不伤 baseline**。

## 7.1 分数标准化

在 `val_known` 上分别把：

* `expr_fused`
* `grn_distance_score`

都转成经验分位数分数：

```text
score_pct(cell) = ECDF_val_known(score)
```

这样两类分数都落在 `[0,1]`，且具有“相对异常程度”含义。

记作：

* `expr_pct`
* `grn_pct`

## 7.2 保守融合公式

我建议用：

```text
dual_fused = expr_pct + λ * max(0, grn_pct - τg)
```

其中：

* `λ = 0.2`
* `τg = 0.90`（即 val_known 上 GRN 分数的 90 分位）

## 7.3 为什么这样融合

这个公式的逻辑是：

* **expression branch 是主判据**
* **GRN 只在“它自己也显示异常”时，才提供加分**
* GRN 不会直接把一个低 expression-risk 的细胞拉成高风险
* 因此比简单平均更不容易稀释原有 expression-only 性能

这非常适合你当前目标：
**先做 non-inferiority + interpretability，而不是盲目追求融合一定涨点。**

## 8. 阈值确定

## 8.1 阈值集

所有阈值都只在 `val_known` 上定。

### expression-only

```text
T_expr = 95th percentile(expr_fused on val_known)
```

### dual fusion

```text
T_dual = 95th percentile(dual_fused on val_known)
```

## 8.2 判别规则

* `score > threshold` → 判为 unknown
* 否则 → 判为最近 known class

这和你之前的思路一致：
验证集里只含 known 时，用高分位数控制已知类误拒率。

## 10. 生物学解释分析

这是这篇工作的核心，不要只停在 ROC。

## 10.1 case study 对象

优先分析三类细胞：

1. **true positive unknown**

   * ASDC 且高 `dual_fused`

2. **false positive known**

   * known cell 但被 dual_fused 拒识

3. **expression-only miss but dual_fused rescue**

   * expression-only 未拒识
   * dual_fused 成功拒识

## 10.2 每个 case study 输出

对每个细胞，输出：

* 最近的 known class
* expression score 各分量
* GRN score
* top deviating regulons
* top TFs
* 与最近 known centroid 的 regulon activity 差异

## 10.3 群体层面图

至少出这 4 张：

1. `regulon activity heatmap`

   * train_known centroids + test_unknown centroid

2. `UMAP`

   * 用 scANVI latent 表示，着色 `dual_fused`

3. `ROC curve`

   * entropy / expr_distance / expr_fused / dual_fused

4. `top regulon barplot`

   * ASDC vs nearest known class

## 12. 成功标准

这轮实验我建议你只看这 3 条：

### 12.1 baseline 成功

`expr_fused` 的 unknown AUROC 维持你之前的高水平

### 12.2 融合非劣

`dual_fused` 相比 `expr_fused`：

* AUROC 不显著下降
* 或下降不超过一个你可接受的小边界

### 12.3 解释有效

至少能从 ASDC 的高分样本中找到**稳定的 regulon/TF 偏离模式**，并且这种偏离不是简单重复 entropy / distance
