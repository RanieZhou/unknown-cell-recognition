## 按顺序运行

所有命令在项目根目录 `D:\Desktop\研一\code\scGRN\` 下执行。

---

### Step 1：scANVI expression backbone

```powershell
python scripts\run_e005_scanvi_backbone.py
```

**做什么**：
- 加载数据，从 `adata.raw.X` 创建 `counts` layer
- 按 70/15/15 stratified 划分 known cells（cDC2/pDC/cDC1），ASDC 全入 test_unknown
- train 中 10% known 标签掩蔽为 `"unlabeled"`（半监督）
- SCVI pretrain (max 200 epochs, early stopping) → scANVI fine-tune (max 100 epochs)
- 计算 entropy + Mahalanobis distance → `expr_fused = 0.5 * entropy_norm + 0.5 * distance_norm`

**预计时间**：10-30 分钟（视 GPU/CPU）

**产物**：
- `outputs/E005/E005_split_summary.csv`
- `outputs/E005/E005_expression_scores.csv` ← 核心
- `outputs/E005/E005_train_known_obs.csv`
- `outputs/E005/E005_val_known_obs.csv`
- `outputs/E005/E005_scanvi_latent_train.npy`
- `outputs/E005/E005_scanvi_latent_test.npy`
- `outputs/E005/scvi_model/` 和 `scanvi_model/`

---

### Step 2：导出 pySCENIC 输入

```powershell
python scripts\run_e005_export_pyscenic_input.py
```

**做什么**：
- 从 Step 1 确定的 `train_known` 细胞中提取 raw count 表达矩阵
- 过滤低表达基因（< 1% 细胞表达）
- 导出为 loom + TSV 双格式

**预计时间**：2-5 分钟

**产物**：
- `outputs/E005/E005_train_known_expr.loom`
- `outputs/E005/E005_train_known_expr.tsv`

---

### Step 3：pySCENIC 三步流程

```powershell
powershell -ExecutionPolicy Bypass -File scripts\run_e005_pyscenic_cli.ps1
```

**做什么**：
1. `pyscenic grn` — 从表达矩阵推导共表达模块（GRNBoost2）
2. `pyscenic ctx` — motif enrichment / pruning
3. `pyscenic aucell` — regulon activity scoring

**预计时间**：30-120 分钟（最耗时，取决于 CPU 核数）

**产物**：
- `outputs/E005/E005_grn_adjacencies.tsv`
- `outputs/E005/E005_regulons.csv`
- `outputs/E005/E005_aucell.loom`

---

### Step 4：GRN space scoring

```powershell
python scripts\run_e005_score_grn_space.py
```

**做什么**：
- 读取 Step 3 的 regulon AUC 矩阵
- 对**所有**细胞（包括 val/test）重新跑 AUCell
- 按 train_known 各类计算 regulon centroid
- 用 cosine distance 计算 `grn_distance_score`

**预计时间**：5-15 分钟

**产物**：
- `outputs/E005/E005_grn_scores.csv` ← 核心
- `outputs/E005/E005_aucell_all_cells.csv`
- `outputs/E005/E005_regulon_centroids.csv`

---

### Step 5：融合 + 评估 + 可视化

```powershell
python scripts\run_e005_dual_fusion.py
```

**做什么**：
- ECDF 归一化 `expr_fused` 和 `grn_distance_score`
- 保守融合：`dual_fused = expr_pct + 0.2 * max(0, grn_pct - 0.90)`
- 95th percentile 阈值判定
- 计算 AUROC / AUPR / FPR95 / accuracy / macro-F1
- 生成 4 张图：ROC curves / UMAP / regulon heatmap / top regulon barplot
- Case study 分析（TP unknown / FP known / dual rescue）
- 输出总结报告

**预计时间**：5-10 分钟

**产物**：
- `outputs/E005/E005_fusion_scores.csv`
- `outputs/E005/E005_metrics.json`
- `outputs/E005/E005_roc.png`
- `outputs/E005/E005_umap_dual_fused.png`

## 常见问题

**Q: Step 1 报 GPU 内存不足？**
scvi-tools 默认用 GPU。如果 GPU 内存不够，在脚本开头加：
```python
scvi.settings.dl_num_workers = 0
```
或设置环境变量 `CUDA_VISIBLE_DEVICES=""` 强制用 CPU。

**Q: Step 3 pyscenic 命令找不到？**
确保 pyscenic 已安装并在 PATH 中。可以用 `python -m pyscenic` 替代直接调用 `pyscenic`。

**Q: Step 4 loom 文件格式不对？**
pySCENIC 版本不同可能导致 loom 输出格式差异。脚本中有自动检测逻辑，如果仍然出错，请反馈具体错误信息。
